01 · System Overview

What Is It?

AQA Agent is an AI-driven test automation system that automatically generates, runs, debugs, and maintains gRPC integration tests for the CypherTrade platform.

Instead of engineers manually writing tests for every new gRPC method, the agent reads the proto contract, reasons about edge cases, writes TypeScript / Playwright test files, runs them, and iterates until they pass — all autonomously.

One-line definition: AQA Agent turns a service name into a passing test suite — end to end — with no human intervention.
02 · Architecture

Core Concept

The agent is built on a deterministic state machine that orchestrates a pipeline of phases. Some phases are pure code (no LLM); others spawn an LLM agent with a specific role, tools, and budget.

Pattern: Deterministic setup → LLM planning → Deterministic guardrails → LLM coding → Deterministic validation → LLM debugging (if needed)

Every phase transition is validated against an explicit PHASE_TRANSITIONS table. Invalid transitions throw immediately — no silent state corruption. Every transition reason is recorded in state.notes for post-mortem audit.

// PHASE_TRANSITIONS — valid paths through the pipeline init → [resolve, failed] resolve → [parse, failed, stopped] parse → [coverage, failed, stopped] coverage → [plan, done, failed, stopped] plan → [plan, implement, save, done, failed, stopped] implement → [implement, validate, plan, save, failed, stopped] validate → [validate, plan, implement, debug, save, done, failed, stopped] debug → [debug, validate, save, failed, stopped] save → [report, failed] report → [done] done / failed / stopped → []

The state machine also tracks agent-to-agent communication. The planner writes schema insights via a save_notes tool call. Those notes travel in state.notes to the coder — explicit, structured, not implicit memory.

03 · Boundaries

Scope & Boundaries

In Scope

  • gRPC integration tests for all services in the skill-trade repo
  • Proto contract parsing and schema validation
  • Test coverage analysis and gap detection
  • Test plan generation (structured, prioritised test cases)
  • Test code writing, execution, and auto-debugging
  • Real-time monitoring via web dashboard (localhost:3000)
  • Cost tracking per run, per agent, per phase
  • Proto snapshot tracking — detect contract changes between syncs

Out of Scope

  • UI / E2E browser tests
  • Performance or load testing
  • Tests for non-gRPC (REST) endpoints
  • Production deployment of generated tests

Key Components

⚙️
Orchestrator
Entry point. Initialises RunState, runs setup phases, dispatches per-method actions, saves ledger in finally block.
🔄
State Machine
Validates all phase transitions against PHASE_TRANSITIONS table. Throws on invalid. Records every reason in state.notes.
🔍
Pipeline Setup
Deterministic phases: resolve → parse → coverage. No LLM cost. Pure TypeScript code.
🎬
Pipeline Actions
Per-action handlers: cover, plan, fix, validate_only, implement_only. Each dispatched from orchestrator switch.
🤖
Agentic Loop
LLM loop per agent call. Extended thinking, tool calls (save_notes, complete_phase), history compression.
📚
Context Builder (RAG)
Assembles system prompts per role: planner vs coder vs debugger. Injects proto, example tests, wrapper, failure patterns.
💰
Cost Tracker
Tracks input / output / cache tokens and USD cost per step, per agent, per phase. Budget limits enforced.
🖥️
Web Dashboard
Single-page app (SSE + localStorage). Pipeline phases, phase inspector, live log, run history, cost, chat.
💾
Memory / Persistence
Project index (coverage %), run history, plan artifacts (resumable runs), run ledger (decisions + facts per run).

Multiple layers prevent silent failures, invalid code, and cost overruns.

CostBudget limit per agentAgentConfig.budgetLimit checked before each LLM call. Run stops cleanly.

07 · Motivation

What Problem Does It Solve?

The skill-trade platform has 8 gRPC services with dozens of methods. Manually writing, maintaining, and running integration tests for all of them is slow, expensive, and error-prone.

ProblemImpactBeforeAfter AQA Agent
Manual test authoringHigh engineer time costHours per methodAutomated
Coverage gapsUntested methods ship to prodUnknown coverageReal-time % per service
Contract driftProto changes break tests silentlyDiscovered in CI failuresProto snapshot diff on every sync
Debugging loopsEngineers debug flaky tests manuallySlow, expensiveAuto-debug with classified failure strategies
Context switchingQA blocks dev velocityQA team bottleneckAgent runs async, unblocks team
Cost visibilityLLM usage untrackedBlack boxUSD cost per run, per agent, per phase
The agent does not replace QA engineers — it eliminates the repetitive, mechanical part of test writing so engineers can focus on complex scenarios, business logic validation, and architecture decisions.
08 · Design

Decisions Taken

Architecture Decisions

DecisionRationale
State machine with explicit transition table Prevents silent phase corruption. Every invalid transition throws with full context. Every reason recorded in state.notes for audit.
Artifact-first types Each phase produces a strongly-typed artifact (NormalizedContract, CoverageReport, TestPlan). Downstream phases receive clean validated data — not raw strings.
History compression LLM context grows linearly with tool calls. Last 2 turns full, older compressed to [compressed: N chars]. Reduces token cost 40–60% on long implement runs.
save_notes as agent-to-agent communication Agents don't share memory implicitly. Planner writes insights explicitly via tool call → travels to coder via state.notes. Structured, auditable, not magic.
Resumable plan / implement split Plan artifacts saved as JSON. implement_only loads them without re-running planner. Allows re-implementing after prompt changes without paying for re-planning.
RAG context builder per agent role Planner prompt ≠ coder prompt ≠ debugger prompt. Each gets only the context it needs. Reduces noise and token waste.
Six run actions Different team needs: analyze-only for reporting, plan-only for review, validate_only for CI, fix for debugging without re-writing.

Tooling Decisions

DecisionRationale
Playwright + TypeScript (not Jest/Mocha)Existing team expertise. gRPC wrappers already built. Fixtures pattern established. No second test runner needed.
Proto files as source of truthGenerated TypeScript code must never be edited manually. Prevents drift between contract and test assumptions.
ts-interface-checker for schema validationRuntime schema checks on every gRPC response. First test in every file validates contract shape. Catches breaking API changes early.
Claude API with adaptive thinkingExtended thinking improves reasoning for complex proto schemas. Thinking tokens not charged at full rate.
SSE for dashboardNo WebSocket server needed. Built into Node.js HTTP. Auto-reconnect on disconnect. Simple and reliable.
09 · Economics

Cost Effectiveness

Model Tier Strategy

TierUsed ForProfile
haikuResearcher, simple queries, context retrievalCheapest — ~$0.0008 / 1K tokens
sonnetPlanner, reviewer, structured reasoningMid — ~$0.003 / 1K tokens
opusCoder (implement), debugger (complex fixes)Most capable — higher cost, best output quality

Cost Reduction Techniques

  • History compression — compresses tool results older than last 2 turns. Reduces context 40–60% on long runs.
  • Prompt caching — system prompts marked for cache. Cache read tokens cost 90% less than input tokens.
  • Budget limits per agent — each AgentConfig has budgetLimit. Agent stops before exceeding it.
  • Per-run budget cap — prevents runaway loops. Default: $10 per run.
  • implement_only action — skips re-planning. Pay for implementation only.
  • analyze action — coverage report only, zero LLM cost.

implementLLM · OpusPer method: reads proto, wrapper, example tests. Writes TypeScript Playwright test file.
validatedeterministicRuns Playwright. Returns TestResult: passed, failed, skipped, errors.
debugLLM · SonnetClassifies failure (A–G). Applies auto-fix or LLM-debug. Re-runs validate.
savedeterministicSaves plan artifacts or last implement run. Updates service index and run history.
reportdeterministicPrints per-method result table. Emits complete SSE event to dashboard.
done / failed / stoppedterminalPipeline ends. Ledger saved regardless (finally block).

Run Actions

ActionWhat It Does
coverFull pipeline: plan → implement → validate → debug. Targets uncovered methods only.
planPlan only. Saves plan artifacts as JSON. No test files written.
implement_onlyLoads saved plans. Implements without re-planning.

Updating Agent Prompts

  • System prompts location: src/rag/context-builder.ts
  • Planner prompt — modify when test plan quality degrades or new test case types needed
  • Coder prompt — modify when generated code style drifts from team conventions
  • Debugger prompt — modify when new failure patterns emerge
  • After prompt change: run implement_only — skips re-planning, applies new coder prompt only
